# Git Max AI: Frontend-Backend Integration Specification

This document outlines the exact data structures, API endpoints, and workflows the Frontend expects from the Backend. Currently, the React frontend renders using static mock data. To integrate the frontend and backend successfully, the backend must implement APIs that return JSON payloads matching the structures defined below.

## 1. Authentication & Workflow Initialization
**Workflow:** 
1. User logs in.
2. Backend returns a JWT or session cookie.
3. Frontend uses the token to authorize subsequent API calls.
4. Frontend fetches global system state (Overall Health Score, Active Agents).

### Recommended Endpoints
- `POST /api/v1/auth/login` - Validates credentials, returns JWT.
- `GET /api/v1/system/health` - Returns the aggregate system health score used on the Repositories and Reports pages.

## 2. Dashboard APIs

### Fetch Risk Trend Evolution
**Endpoint:** `GET /api/v1/analytics/risk-trends`
**Description:** Supplies the time-series data for the "Risk Trend Evolution" area chart.
**Expected Output:**
```json
[
  { "day": "Mon", "risk": 42, "resolved": 38 },
  { "day": "Tue", "risk": 38, "resolved": 35 }
  // ... continuous array
]
```

### Fetch Top Risky Repositories
**Endpoint:** `GET /api/v1/repositories/risky`
**Description:** Populates the table prioritizing repos that require immediate attention.
**Expected Output:**
```json
[
  { 
    "name": "legacy-auth-service", 
    "risk": "high", // 'high' | 'medium' | 'low'
    "score": 87, 
    "issues": 14, 
    "trend": "up", // 'up' | 'down' | 'stable'
    "lang": "TypeScript" 
  }
]
```

### Fetch AI Agent Activity Stream
**Endpoint:** `GET /api/v1/agents/activity`
**Description:** Real-time or polled stream of what the autonomous agents are doing.
**Expected Output:**
```json
[
  {
    "agent": "Repo Agent",
    "icon": "smart_toy", // Material Symbol mapping
    "color": "primary", // 'primary' | 'secondary' | 'tertiary' | 'error'
    "message": "Scanning api-v3-internal for structural drift.",
    "time": "2m ago",
    "status": "active" // 'active' | 'idle'
  }
]
```

## 3. Repositories Connectivity

### Fetch Connected Repositories
**Endpoint:** `GET /api/v1/repositories`
**Description:** Returns the active repositories connected to Git Max AI.
**Expected Output:**
```json
[
  {
    "name": "core-engine",
    "org": "precision-ai/core",
    "branch": "main-production-branch",
    "health": 92, // 0-100 score
    "risk": "low", // 'high' | 'medium' | 'low' | 'stable'
    "lastSync": "14 minutes ago", // or ISO Timestamp
    "version": "v4.2.0-stable",
    "lang": "TypeScript",
    "stars": 842
  }
]
```

**Note:** The Frontend will also require `POST /api/v1/repositories/sync` endpoints to trigger manual webhooks for Gitlab/Github ingestion.

## Important Workflow Considerations

1. **Websockets vs Polling:** The AI Agent stream on the Dashboard currently relies on static data. For a truly "real-time" feel, consider implementing a WebSocket connection (`wss://api.domain.com/v1/stream`) to emit events whenever an agent scans a repo.
2. **Date Formatting:** The frontend currently expects pre-formatted human-readable strings like `"2h ago"`. It is better for the backend to return standard **ISO 8601 Timestamps** (e.g. `2024-10-15T14:30:00Z`) and let the Frontend convert them using libraries like `date-fns`.
3. **Pagination:** Tables (like the Active Repositories and PRs) do not currently have pagination. The Backend should implement standard offset/limit logic (`?page=1&limit=20`) and the frontend table views will need state updates to support `Next/Prev` fetching.
